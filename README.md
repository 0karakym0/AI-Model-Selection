# MWS Model Selection Assistant

Ассистент для клиентов MWS, помогающий выбрать подходящую GPT-модель из каталога GPT Model Hub под продуктовый кейс и бюджет пользователя.

Реализован на **Google Agent Development Kit (ADK)** с **OpenAI-совместимым API** для интеграции.

## Краткое описание задачи

Сервис принимает от пользователя описание кейса (тип нагрузки, формат запросов, объём трафика, требования к качеству, бюджет), анализирует актуальные данные о моделях и ценах с сайта MWS, подбирает подходящие модели и выдаёт структурированный отчёт с расчётом стоимости.

## Архитектура

### Агентский паттерн: Single Agent with Tools

Выбран архитектурный паттерн **Single Agent with Tools** — один агент, оснащённый инструментами для получения данных и расчёта стоимости. Агент выполняет роль эксперта-консультанта и самостоятельно принимает решение о вызове инструментов в зависимости от запроса пользователя.

**Роли агента:**

- **MWS Assistant Agent** — единственный агент в системе. Отвечает за:
  - Получение актуального каталога моделей и тарифов через вызов `get_actual_prices()`
  - Анализ кейса пользователя и подбор подходящих моделей
  - Расчёт стоимости через вызов `calculate_mws_cost()`
  - Формирование структурированного отчёта из четырёх блоков
  - Поддержку диалога в рамках сессии

**Схема взаимодействия:**
```
Пользователь
│
▼
FastAPI (/v1/chat/completions)
│
▼
LlmAgent (Google ADK)
│
├── get_actual_prices() ──► HTTP-запросы к mws.ru/docs
│
└── calculate_mws_cost() ─► Расчёт стоимости по тарифам
```


## Структура проекта

| Файл | Назначение |
|------|-----------|
| `server.py` | FastAPI сервер, эндпоинт `/v1/chat/completions` (OpenAI-совместимый API) |

| `tests/` | Автотесты (калькулятор + парсер) |

## Реализованные требования

### Функциональные

- OpenAI-совместимый API: эндпоинт `POST /v1/chat/completions`
- Формат запроса и ответа соответствует спецификации OpenAI Chat Completions API
- Агент на Google ADK с архитектурным паттерном Single Agent with Tools
- Динамический парсинг данных о моделях и ценах с сайта MWS (без хардкода)
- Расчёт стоимости с учётом отпускных единиц и округления вверх
- Структурированный отчёт из четырёх блоков: Входные данные, Рекомендованные модели, Расчёт стоимости, Пояснения/ограничения
- Поддержка диалога: уточнение вводных, ответы на дополнительные вопросы в рамках сессии

### Бонусные

- Валидация входных данных через Pydantic (ограничение ролей, проверка на пустые сообщения, проверка что последнее сообщение от пользователя)
- Observability: логирование ключевых шагов, метрики времени ответа и количества запросов к сайту MWS
- Автотесты: 10 тестов для калькулятора стоимости и парсера данных

## Пример запроса и ответа API

### Запрос

```bash
curl -X POST http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mws-agent",
    "messages": [
      {"role": "user", "content": "Мне нужно написать диплом, примерно 50 страниц. Бюджет до 200 рублей. Какие модели подойдут?"}
    ]
  }'
```

## Инструкция по локальному запуску
### Предварительные требования
- Python 3.10+

- Google Cloud SDK — установите и выполните аутентификацию:
```bash
gcloud auth application-default login
```
- Vertex AI API — должен быть включён в вашем Google Cloud проекте

- Google Cloud Project — создайте проект в Google Cloud Console

### Переменные окружения
Создайте файл .env в корне проекта на основе .env.example:
```ini
GOOGLE_CLOUD_PROJECT=your-project-id
GOOGLE_CLOUD_LOCATION=us-central1
```
### Быстрый старт
```bash
# 1. Клонирование репозитория
git clone <repo-url>
cd AI-Model-Selection

# 2. Создание файла окружения
copy .env.example .env   # Windows
# cp .env.example .env   # Linux/MacOS

# 3. Отредактируйте .env — укажите ваш GOOGLE_CLOUD_PROJECT

# 4. Создание виртуального окружения
python -m venv venv

# 5. Активация виртуального окружения
venv\Scripts\activate     # Windows
# source venv/bin/activate  # Linux/MacOS

# 6. Установка зависимостей
pip install -r requirements.txt

# 7. Запуск сервера
python server.py
```
Сервис будет доступен на http://localhost:8000.

Swagger UI: http://localhost:8000/docs

### Запуск тестов
```bash
pytest tests/ -v
```
10 тестов проверяют корректность расчётов стоимости и структуру данных парсера.

### Ограничения
- Данные парсятся динамически — цены и модели всегда актуальны, не хардкодятся

- Оценка токенов — если пользователь не указывает точное количество токенов, агент оценивает их самостоятельно с пояснением метода

- 24-часовой остаток — в расчёте стоимость округляется вверх до целой отпускной единицы, моделируя наихудший сценарий без учёта накопления остатка между запросами

- Валюта — все цены указаны в российских рублях (₽)

- Google Cloud Project — каждый пользователь должен использовать собственный проект с включённым Vertex AI API
